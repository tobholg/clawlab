# 2026-01-31 — ClawLab External Stakeholders Feature

## Major Build Session

Built the complete External Stakeholders system for ClawLab across 6 phases:

### Completed Phases
1. **Data models + migrations** — ExternalSpace, StakeholderAccess, InformationRequest, ExternalTask, ExternalComment, AIContextDoc
2. **Space management (owner side)** — CRUD, invite links, stakeholder management with per-user task permission toggle
3. **Stakeholder portal** — Magic link signup, dashboard, portal with Overview/My Requests/Q&A tabs
4. **Inbound queue** — Unified view for owners to manage incoming tasks and IRs, accept/reject/respond
5. **AI chat integration** — Stakeholder chat with project context, IR/task drafting with approval UI
6. **Portal redesign** — GitHub-style activity heatmap, hierarchical task display, animations

### Key Design Decisions
- **External Spaces** = isolated portals per project (e.g., one for client, one for investors)
- **AI intermediary** = stakeholders chat with AI that knows project context, escalates to team via Information Requests
- **Comments work on both tasks AND IRs** — shared conversation between stakeholders and team
- **Per-stakeholder permissions** — owners toggle `canSubmitTasks` individually
- **Task submission via chat** — AI drafts tasks from natural language (e.g., paste an email, get a bug report)

### Technical Notes
- ExternalComment model supports both `externalTaskId` and `informationRequestId` (one must be set)
- Added `isTeamMember` flag to distinguish team vs stakeholder comments
- Activity tracker uses blue color palette (same as workspace)
- Subtasks collapsed by default in stakeholder portal
- Inbound queue shows Pending | All | Resolved tabs
